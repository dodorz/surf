# 通用开发规范

## 版本管理

- 遵循语义化版本规范 (Semantic Versioning): `MAJOR.MINOR.PATCH`
- 每次功能修复提交递增 PATCH 版本号
- 新功能提交递增 MINOR 版本号
- 不兼容的改动递增 MAJOR 版本号
- Git 提交信息使用英文

## 依赖管理

- 使用现代包管理器（如 `uv`）
- 依赖版本锁定文件不加入版本控制（如 `uv.lock`），允许不同环境自动重建依赖

## 命令行参数设计

- **简洁优先**：提供短参数（如 `-o`）作为长参数的快捷方式
- **避免冗余**：相同功能的参数只保留一个入口
- **一致性**：短参数与长参数保持语义一致

**代理参数示例**：
- `-x/--proxy auto`：使用环境变量代理（默认）
- `-x/--proxy win`：使用 Windows 系统代理（WinINET）
- `-x/--proxy no`：不使用代理
- `-x/--proxy set`：使用自定义代理（配合 `--set-proxy` 或配置文件）

## 文档同步

后续任何代码更改，如涉及命令行参数、功能特性或配置文件的修改，应同步更新以下文档：

1. **`--help` 输出**：确保帮助信息与实际参数一致
2. **`README.md`**：用户文档需反映最新功能
3. **项目需求文档**（如 `PROMPT.md`）：功能需求和接口说明需同步更新

## 通用配置

- 敏感配置信息通过环境变量或独立配置文件管理
- 提供配置文件模板（如 `config.ini.example`）供用户参考
